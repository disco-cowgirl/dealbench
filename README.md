# DealBench — your site, rebuilt & owned by you

Full working copy of dealbench.buildwithlocus.com, extracted from the Locus
dashboard on July 15, 2026 — with the bug that broke sign-up **fixed** (the
Locus deployment never connected its database; this version reads a standard
`DATABASE_URL` and works anywhere).

## What's in here

- `frontend/` — the React site (all pages: Home, Signup, Signin, Log a deal,
  Privacy, FAQ, 404). Same dark "auditor with a spreadsheet" look.
- `backend/` — the Express API (accounts, sessions, anonymous deal
  submissions, the live median feed).
- `Dockerfile` — builds both into one deployable container.
- `render.yaml` — one-click deploy blueprint for Render.com (free tier),
  including a free Postgres database wired in automatically.

## Checkpoint 2 instructions (do with Claude, or follow solo)

1. Create a free GitHub account (github.com) if you don't have one, and a
   free Render account (render.com) — sign up with GitHub.
2. Push this folder to a new GitHub repo called `dealbench`.
3. In Render: **New + → Blueprint**, pick the `dealbench` repo. Render reads
   `render.yaml`, creates the web service + free Postgres, and deploys.
4. Site goes live at `https://dealbench.onrender.com` (rename in settings).
5. Test: create an account on the live site. It will work — the database is
   actually connected this time.

## Changes vs. the Locus version (all deliberate)

- `backend/src/db.ts`: reads `DATABASE_URL` (standard everywhere) with SSL
  for managed Postgres; still accepts Locus's `pg_DATABASE_URL` as fallback.
- "See the Numbers" buttons: were Locus checkout-widget buttons; now route
  to `/signup`. To charge the $16.99 access fee off-platform, add a Stripe
  Payment Link later (5-minute job — ask Claude).
- `frontend/index.html`: removed Locus analytics/attribution scripts.
- Removed the "Built and Run by Locus Founder" footer bar.

## Notes

- Signups store emails + bcrypt-hashed passwords in Postgres. Sessions are
  30-day JWT cookies. `SESSION_SECRET` is auto-generated by render.yaml.
- Local dev: `cd backend && npm install && npm run dev` (needs a local
  DATABASE_URL), then `cd frontend && npm install && npm run dev`.
